3. FUNCIONES SQL SERVER SIN CONVERTIR EN VISTAS (Múltiples líneas)
   ────────────────────────────────────────────────────────────────────────

   a) ISNULL() - FUNCIÓN SQL SERVER, NO EXISTE EN PostgreSQL
      Ubicaciones: Líneas 62, 63, 89, 90 (varias ocurrencias)

      EJEMPLOS:
      LÍNEA 62: ISNULL(A.apellido1, '')
      LÍNEA 62: ISNULL(A.apellido2, '')
      LÍNEA 63: ISNULL(A.ext, '')

      ERROR EN PostgreSQL: "ERROR: function isnull() does not exist"
      SOLUCIÓN: Reemplazar ISNULL(x, y) por COALESCE(x, y)

   b) CONCATENACIÓN CON + (SQL Server specific)
      Ubicaciones: Líneas 62-63, 89-90

      EJEMPLOS:
      LÍNEA 62-63: ISNULL(A.apellido1, '') + ' ' + ISNULL(A.apellido2, '') + ' ' + ISNULL(A.nombres, '')
      LÍNEA 63: A.id_afi + ' - ' + ISNULL(A.ext, '')

      ERROR EN PostgreSQL: "ERROR: operator does not exist: character varying + character varying"
      SOLUCIÓN: Reemplazar todos los + por ||

      REESCRITURA COMPLETA LÍNEA 62-63:
      ACTUAL:
      ISNULL(A.apellido1, '') + ' ' + ISNULL(A.apellido2, '') + ' ' + ISNULL(A.nombres, '') AS NOMBRE,
      A.id_afi + ' - ' + ISNULL(A.ext, '') AS CI

      CORRECTO:
      COALESCE(A.apellido1, '') || ' ' || COALESCE(A.apellido2, '') || ' ' || COALESCE(A.nombres, '') AS NOMBRE,
      A.id_afi || ' - ' || COALESCE(A.ext, '') AS CI

3. PROBLEMAS CON TIPOS DE DATOS (Inconsistencias)
   ────────────────────────────────────────────────────────────────────────

   a) TablaAfiliadosAux (líneas 368-385) - DOUBLE PRECISION para IDs
      ACTUAL:
      id_fed DOUBLE PRECISION,
      id_cent DOUBLE PRECISION,
      id_sind DOUBLE PRECISION,
      id_cato DOUBLE PRECISION,

      PROBLEMA:
      - DOUBLE PRECISION es punto flotante (impreciso para enteros grandes)
      - Perderá precisión en números > 2^53
      - Imposible hacer JOINs con BIGINT (tipo diferente)
      - "Trabajaría" pero datos serían inexactos

      CORRECTO: Deberían ser BIGINT
      IMPACTO SEVERO: JOINs entre TablaAfiliadosAux e id_fed (BIGSERIAL) fallarán

   b) ControlCatoBitacora sup_mensura - Tipo inconsistente
      ControlCatoBitacora línea 330: sup_mensura DOUBLE PRECISION
      ControlCato línea 41: sup_mensura decimal

      PROBLEMA: Comparaciones entre tipos diferentes funcionan pero son ineficientes
      IMPACTO: Queries lentas, posibles imprecisiones redondas
